class mubosym_core.MBSworld(name='', connect=False, force_db_setup=False)¶

All storages lists and dictionaries for a full multibody sim world. Keeps track of every frame, body, force and torque. Includes also parameters and external model interface (e.g. tire).

add_body_3d(new_body_name, str_n_marker, mass, I, joint, parameters=[], graphics=True)¶

Core function to add a body for your mbs model. Express the pos and vel of the body in terms of q and u (here comes the joint crunching). Generalized coordinates q and u are often (not always) written in the rotated center of mass frame of the previous body.

Parameters:
  • new_body_name (str) – the name of the new body (freely given by the user)
  • str_n_marker (str) – the name string of the marker where the new body is related to (fixed on)
  • mass (float) – the mass of the new body
  • I (float*3) – the inertia of the new body measured in the body symmetric center of mass frame (float*3 list) [Ixx, Iyy, Izz]
  • joint (str) – the type of the joint: possible choices are (see code and examples). You can add whatever joint you want to. Here the degrees of freedom are generated.
  • parameters (float*n) – the parameters to descripe the joint fully (see code)
  • graphics (str) – the choice of the grafics representative (ball, car, tire)

add_geometric_constaint(str_m_b, equ, str_m_b_ref, factor)¶

Function to add a geometric constraint (plane equation in the easiest form)

Parameters:
  • str_m_b (str) – name of the body/marker for which the constraint is valid
  • equ (sympy-expression) – is of form f(x,y,z)=0,
  • str_m_b_ref (str) – reference frame
  • factor (float) – the factor of the constraint force (perpendicular to the plane). If this is higher, the constraint is much better fullfilled, but with much longer integration time (since the corresponding eigenvalue gets bigger).
add_marker(new_marker_name, str_n_related, vx, vy, vz, phix=0.0, phiy=0.0, phiz=0.0)¶

Add a fixed marker framer related to a body frame. Marker frames are used to add joints or forces (they usually act between two of them or a body and a marker).

Parameters:
  • new_marker_name (str) – the name of the new marker
  • str_n_related (str) – the reference fixed body frame
  • vx, vy, vz (float) – the const. translation vector components in the related frame (float*3)
  • phix,phiy,phiz (float) – the const. rotation Eulerian angles for a new (body-fixed) orientation of the marker frame

class one_body_force_model_interface.one_body_force_model¶

a one body force model consists of:

  • coordinate trafo generalized coords -> body coordinates (denoted list of) including pos, vel, orientation, and omega
  • force calculator given as a python function with input according to our interface
  • some preparation function: lambdifier to include symbolic functions into lambdas
calc(inp)¶

the python function which connects some external model calculation with the mbs model e.g. tire-model, rail model

  • input list inp are some relevant model coordinates (out of 12 possible): [ x, y, z, nx, ny, nz, x_pt, y_pt, z_pt, omega_x, omega_y, omega_z ] = inp
  • output list is force in cartesian coord. world and torque cartesian coord. world
force_lam(w)¶

the model force/torque via lambdified expressions, input parameter here is always the full state vecor t,q,u Output is the force/toque via the model calc-function the nested input for the calc routine is fully possible written out:

  • self.lam_t, self.lam_x, self.lam_y, self.lam_z,
  • self.lam_nx, self.lam_ny, self.lam_nz,
  • self.lam_x_pt, self.lam_y_pt, self.lam_z_pt,
  • self.lam_omega_x self.lam_omega_y, self.lam_omega_z

but can be reduced to a subset
